# Vue 速查表来源策略

## 顺序

1. 先检查当前仓库的 `AGENTS.md`、README、配置、现有指南、速查表和真实代码。
2. 再使用官方文档核对版本敏感行为。
3. 官方文档优先于博客、社区文章、旧教程和模型记忆。
4. 本地材料与官方文档冲突时，采用官方文档并标记本地材料已过时。
5. 无法访问所需官方页面时，加入 `Verification Needed`，不要给出未经核对的确定结论。

## 官方来源

1. Vue official docs: `https://vuejs.org/`
2. Vue TypeScript guide: `https://vuejs.org/guide/typescript/overview`
3. Vue Router official docs: `https://router.vuejs.org/`
4. Pinia official docs: `https://pinia.vuejs.org/`
5. Vite official docs: `https://vite.dev/`
6. Vitest official docs: `https://vitest.dev/`
7. Vue Test Utils official docs: `https://test-utils.vuejs.org/`
8. TypeScript Handbook / TSConfig Reference: `https://www.typescriptlang.org/docs/` 与 `https://www.typescriptlang.org/tsconfig/`
9. Nuxt official docs: `https://nuxt.com/docs/`，仅在相关时使用

## 使用要求

- 记录实际打开的 URL，不把来源列表当成已验证证据。
- 对 macro、Router、Pinia、testing、Vite、`vue-tsc` 和 Nuxt behavior 核对当前页面。
- 速查表末尾列出实际使用的官方来源。
