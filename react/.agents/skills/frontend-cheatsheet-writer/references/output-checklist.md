# Output Checklist

Before finishing, verify:

- The output is a cheatsheet, not a full tutorial.
- The scope is clear.
- Long standalone cheatsheets include `## 目录` near the top.
- If present, `## 目录` appears before `## 0. Scope`.
- If present, `## 目录` matches the actual cheatsheet headings and does not import full chapter guide-only sections.
- Core concepts are reduced to accurate one-sentence summaries.
- JavaScript runtime and TypeScript type system are separated.
- Framework behavior and language mechanism are separated.
- API tables are complete for the requested scope.
- Similar concepts are compared with actual differences.
- Common mistakes include wrong code, error type, violated rule, correction, and recognition method.
- Minimal templates are practical and copyable.
- Official documentation links are included.
- Real file paths in code-window title bars match a nearby project or template structure when such paths are used.
- Conceptual examples, mistakes, contrasts, and reusable patterns use logical `Snippet:` or `Template:` title-bar names instead of fake real-looking paths.
- Code and code comments contain no Chinese characters.
- Every code example uses the macOS-style HTML title bar plus a normal Markdown fenced code block.
- Every title bar has red/yellow/green dots.
- Every title bar displays a file name, logical snippet name, `Terminal`, `Output`, or `Error`.
- Decorative UI elements are outside the actual source code.
- Source code remains clean, copyable, and executable.
- No real source code is placed inside raw HTML `<pre><code>`.
- Correct fenced code language identifier is used: `ts`, `tsx`, `js`, `jsx`, `bash`, `json`, `html`, `css`, or `txt`.
- CSS includes `.macos-code-titlebar + pre` when local styling is included.
- The final response includes a self-check checklist.
