# Front-end Learning Lab

This repository records my front-end learning process from fundamentals to real-world engineering.

It is not a random collection of exercises. It is organized as a long-term learning lab for JavaScript, TypeScript, React, Node.js fundamentals, tooling, and small projects.

## Current Focus

- JavaScript language mechanisms
- Functions, scope, closure, prototype, and `this`
- TypeScript type system fundamentals
- React component architecture
- Node.js runtime fundamentals for front-end engineering
- Git, GitHub, package management, formatting, and linting

## Repository Structure

| Directory | Purpose |
| --- | --- |
| `javascript/` | JavaScript fundamentals and mechanism practice |
| `typescript/` | TypeScript type system practice |
| `react/` | React component and application practice |
| `nodejs/` | Node.js runtime fundamentals |
| `mini-projects/` | Small front-end projects and feature demos |
| `notes/` | Learning notes, summaries, and review materials |

## Learning Order

```text
JavaScript fundamentals
  -> TypeScript type system
  -> React application architecture
  -> Node.js fundamentals
  -> Next.js full-stack front-end projects
  -> Production-level engineering practices
```

## Naming Rules

Exercise files should use descriptive English names.

Good examples:

```text
function-declaration.js
arrow-function.js
scope-chain.js
closure-counter.js
prototype-lookup.js
array-sort-compare.js
object-assign-spread.js
```

Avoid unclear names:

```text
1.js
2.js
test.js
demo.js
new.js
aaa.js
```

## Commit Strategy

Each commit should describe one clear learning step.

Good examples:

```text
Add JavaScript function declaration examples
Add array sorting comparison practice
Add notes for object copy mechanisms
Add React props practice
```

## Goal

The goal of this repository is to build a strong front-end engineering foundation before moving into larger portfolio projects.
